# Contributing to Pickle Rick Grok

Thank you for your interest in contributing to Pickle Rick Grok!

## Code of Conduct

We expect all contributors to behave respectfully and professionally.

## How to Contribute

### 1. Contributor License Agreement (CLA)

**All contributions require signing our CLA.**

Before we can accept your pull request, you must sign the Contributor License Agreement. This gives the project maintainers the flexibility to license the code under additional terms in the future (for example, offering a commercial version) while still keeping the project open source under the Apache 2.0 License.

When you open a pull request, the [CLA Assistant](https://cla-assistant.io) bot will automatically prompt you to sign the CLA if you haven't already.

> **Note for maintainers**: To enable CLA signing, go to https://cla-assistant.io and authorize it on this repository. Use the Apache 2.0 CLA template.

### 2. Development Process

- All significant work should start with a **machine-verifiable PRD**.
- We strongly prefer using the project's own tools (`/pickle-prd`, `/pickle-refine-prd`, `/pickle-pipeline`, etc.) to develop the project.
- Large changes should be broken into atomic, reviewable tickets.
- After implementation, run the system's own quality and review tools (Anatomy Park, Szechuan Sauce, Citadel, etc.).

### 3. Pull Request Process

1. Fork the repository and create your branch from `main`.
2. Make your changes.
3. Ensure your code follows the project's style and passes any existing tests/linters.
4. Open a Pull Request against the `main` branch.
5. Sign the CLA when prompted by the CLA Assistant bot.
6. Request a review.

We may ask you to rebase or squash your commits before merging.

### 4. Reporting Bugs / Requesting Features

Please open an issue with as much detail as possible. For feature requests, consider writing a short PRD using the project itself first.
